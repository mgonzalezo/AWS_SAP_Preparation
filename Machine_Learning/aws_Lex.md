# Amazon Lex

- Service for building conversational interfaces (chatbots) into applications using voice and text.
- Create chatbots to use as Interactive Voice Response (IVR) systems.
- Provides advanced deep learning functionalities of automatic speech recognition (ASR) for speech-to-text.
- Natural language understanding (NLU) to recognize intent of the text.

## Amazon Lex V2 (Current Version)

Amazon Lex V2 is the current version with enhanced features and capabilities.

**⚠️ Lex V1 Deprecation Timeline**:

- **March 31, 2025**: No new Lex V1 resources can be created.
- **September 15, 2025**: All Lex V1 API requests will fail.
- **Migration required**: Migrate existing Lex V1 bots to Lex V2.

## Key Features

- Build sophisticated conversational bots with multi-turn conversations.
- Automatic speech recognition (ASR) converts speech to text.
- Natural language understanding (NLU) recognizes user intent.
- Integration with AWS Lambda for fulfillment logic.
- Pre-built integrations with messaging platforms (Facebook, Slack, Twilio).
- Multi-language support for global applications.

## Use Cases

- **Contact center IVR**: Automate customer service calls with voice-enabled bots.
- **Customer support chatbots**: Handle common questions on websites and mobile apps.
- **Enterprise productivity**: Schedule meetings, book travel, access information.
- **IoT devices**: Voice control for smart home and connected devices.

## Integration with AWS Services

- **Amazon Connect**: Build intelligent contact center solutions.
- **Amazon Kendra**: Enhance bots with enterprise search capabilities.
- **AWS Lambda**: Execute business logic and integrate with backend systems.
- **Amazon Polly**: Text-to-speech for voice responses.
- **Amazon Comprehend**: Analyze sentiment and extract insights from conversations.

## Exam Tips

- Amazon Lex is the managed service for building conversational interfaces.
- Lex V1 is deprecated - use Lex V2 for new applications (2025).
- Integrates with Amazon Connect for contact center automation.
- Lambda functions handle fulfillment logic and backend integration.
- Supports both voice (ASR) and text (NLU) interactions.
- Pre-built integrations reduce development time for common platforms.
